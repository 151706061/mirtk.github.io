43 /**
44  * Registered point set
45  *
46  * A registered point set is aligned with another data set (i.e., an image or
47  * another point set) through a given transformation. If no transformation is
48  * set (i.e., NULL), the point set is fixed and usually serves as target for
49  * the registration. Given the pull-back character of image transformations,
50  * the point set which is being transformed is generally the one defined in
51  * the space of the target image (i.e., the fixed reference image). We therefore
52  * refer to the transformed point set as the "target" instead, whereas the
53  * "source" point set is usually not transformed. Eventually, however, both
54  * target and source point sets may be transformed, e.g., in case of a
55  * symmetric registration.
56  *
57  * Besides the geometry, i.e., point positions, a point set may also have a
58  * topological structure as in case of a tetrahedralized simplicial complex or
59  * a boundary surface mesh. Point set distance measures and constraints may be
60  * specialized for surface meshes only, such as the surface curvature constraint,
61  * for example (CurvatureConstraint). These objective function terms only
62  * consider the surface of a given point set which is provided by this class.
63  * If the input point set itself is already a vtkPolyData object, the point set
64  * and its surface are identical. Otherwise, an instance of this class extracts
65  * the surface of the point set using the vtkDataSetSurfaceFilter upon
66  * initialization and keeps the point positions and data of the registered point
67  * set and its corresponding surface in sync.
68  *
69  * \note VTK does not make use of the const keyword and none of the member
70  * functions are declared const. Therefore, returning a pointer to a
71  * const VTK object is useless. Thus, this class grants anyone non-const
72  * access to the internal VTK data objects even if these are not to be
73  * modified by the caller.
74  */
