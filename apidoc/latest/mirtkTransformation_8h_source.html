47 /**
48  * Abstract base class for general transformations.
49  *
50  * This is the abstract base class which defines a common interface for all
51  * transformations. Each derived class has to implement at least the abstract
52  * methods and some of the virtual ones. Most other methods call these virtual
53  * methods and should not be required to be overwritten in subclasses.
54  *
55  * The second time argument to the interface methods corresponds to the time
56  * of the untransformed source image. It is only considered by some 3D+t
57  * transformations, in particular those which parameterize the transformation
58  * using a non-stationary velocity field.
59  */

521  /// to a gradient w.r.t the parameters of this transformation.
522  ///
523  /// If the transformation itself is non-parametric, the gradient will be passed through
524  /// unchanged. The default implementation uses the full Jacobian matrix computed for each
525  /// DoF separately (i.e., calls JacobianDOFs for each DoF).
526  ///
527  /// For 4D transformations, the temporal coordinate t used for the computation of the Jacobian
528  /// of the transformation w.r.t the transformation parameters for all spatial (x, y, z) voxel
529  /// coordinates in world units, is assumed to correspond to the temporal origin of the given
530  /// gradient image. For 4D transformations parameterized by velocities, a second time for the
531  /// upper integration bound can be provided as last argument to this method. This last
532  /// argument is ignored by transformations parameterized by displacements.
533  ///
534  /// \sa ImageSimilarityMetric::EvaluateGradient

747 // -----------------------------------------------------------------------------
748 inline bool HaveSameDOFs(const Transformation *t1, const Transformation *t2)
749 {
750  // If the virtual HasSameDOFsAs function was not overriden by the specialized
751  // type of one transformation, it is assumed that it uses the _Param memory to
752  // store its transformation parameters. However, the other transformation
753  // might be a specialized type which does not make use of it. In this case,
754  // even if the two _Param pointers do not reference the same memory, the two
755  // transformations may still use the same parameters because the other
756  // transformation directly or indirectly wraps this transformation. Therefore,
757  // the check with the transformations exchanged and the boolean OR (not AND).
758  return t1->HasSameDOFsAs(t2) || t2->HasSameDOFsAs(t1);
759 }
