42  * Adaptive local remeshing of triangulated surface mesh
43  *
44  * Park et al., A non-self-intersecting adaptive deformable surface for
45  * complex boundary extraction from volumetric images, 25, 421â€“440 (2001).
46  *
47  * \todo Interpolate cell data during remeshing. The current implementation only
48  * preserves and interpolates point data arrays. Cell attributes are discarded.

132  /// Point data array used to adapt the edge length range for each node
133  ///
134  /// The scalar point data values are rescaled linearly to [0, 1] after
135  /// clamping the point data range to the 5th and 95th percentile range.
136  /// The rescaled value is then plugged into a logistic function which
137  /// determines the linear interpolation weights of the global
138  /// _MinEdgeLength and _MaxEdgeLength range. This obtains an individual
139  /// edge length range for each point. The desired edge length range of a
140  /// given edge is then the mean of the minimum/maximum edge length of the
141  /// two end points of the edge.
142  mirtkPublicAttributeMacro(vtkSmartPointer<vtkDataArray>, AdaptiveEdgeLengthArray);
143

244  /// Check connectivity of edge neighbors
245  ///
246  /// If other vertex of edge neighbor triangle has connectivity three,
247  /// the three triangles adjacent to it are replaced by the union of these
248  /// triangle and the melting operation can be performed. Otherwise, if
249  /// more than one node with connectivity three is found which is adjacent
250  /// to the edge corners, melting the (triangle) edge would cause degeneration
251  /// of adjacent triangles.
252  ///
253  /// @return ID of other vertex of edge neighbor or -1 if not unique or if
254  /// its connectivity prohibits a melting operation.
255  vtkIdType GetCellEdgeNeighborPoint(vtkIdType, vtkIdType, vtkIdType, bool = false);
