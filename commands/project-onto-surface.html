project-onto-surface¶

Synopsis¶

project-onto-surface <input> <output> -image <file> [options]
project-onto-surface <input> <output> -labels <file> [options]
project-onto-surface <input> <output> -boundary [-nolabel-points] [-nolabel-cells] [-name <scalars>]

Description¶

Assign scalars or labels to either the vertices or the cells of a surface mesh. When the input as a real-valued image, the values are linearly interpolated. When the input is a segmentation labels image, the value assigned to a vertex/cell is the label of the nearest voxel in the given segmentation image. For the projection of cortical labels onto the WM/cGM or cGM/CSF boundary, use project-onto-surface -white or project-onto-surface -pial.

Arguments¶

input¶

Input surface mesh.

output¶

Output surface mesh.

Command options¶

-image <file>¶

Input real-valued scalar/vector image.

-labels <file>¶

Input segmentation image with positive integer labels.

-name <name>¶

Name of output scalar array. (default: Scalars or Labels)

-celldata, -nocelldata¶

Assign values to cells of input surface. (default: off)

-pointdata, -nopointdata¶

Assign values to points of input surface. (default: on)

-write-dilated-labels <file>¶

Write image of dilated labels.

-pial¶

Input surface is cortical cGM/CSF boundary.

-white¶

Input surface is cortical WM/cGM boundary.

-min-size <n>¶

Surface patches with less than n points are removed. (default: 0)

-fill, -nofill¶

Fill holes in projected surface parcellation.

Standard options¶

-v, -verbose [n]¶

Increase/Set verbosity of output messages. (default: 0)

-debug [level]¶

Increase/Set debug level for output of intermediate results. (default: 0)

-version [major.minor]¶

Print version and exit or set version to emulate.

-revision¶

Print revision (or version) number only and exit.

-h, -help¶

Print help and exit.
